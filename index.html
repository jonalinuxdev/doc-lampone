What is Lampone?

Lampone is an advanced, browser-based IPTV media player designed to offer a fully client-side solution for streaming and organizing television content using standard .m3u and .m3u8 playlists. It is built with a strong focus on usability, speed, compatibility, and transparency.

The project is ideal for individuals, organizations, or broadcasters who want a modern and modular interface to deliver live or on-demand TV channels without depending on proprietary apps or heavy backend systems. Lampone runs entirely in the browser and is compatible with any device supporting HTML5 and MediaSource Extensions.

▲ Example of Lampone running a sample M3U playlist with quality selection, flags, and EPG interface

Core Features

  • Zero-Server Architecture: Everything happens in the client’s browser—no backend, no database.
  • Instant Channel Loading: Upload or link playlists, and Lampone automatically parses, validates, and renders channels grouped by country or content category.
  • Persistent User Experience: All user data (last loaded playlists, selected groups, etc.) is stored in localStorage so the environment is restored on refresh or return visits.
  • Real-Time Filtering: Built-in search bar lets users find channels instantly by name or group.
  • Intelligent Group Navigation: Channels are classified by the group-title tag and accessible through flag icons and tags in a right-side scrollable menu.
  • HLS Adaptive Streaming: Uses HLS.js to provide bufferless, high-quality playback with automatic resolution scaling based on bandwidth and playback stats.
  • Program Guide (EPG): Integrates XMLTV data sources and dynamically matches current/next programs via tvg-id or tvg-name.
  • Clear User Controls: One-click reset to wipe stored data and return the app to its default state.

Technological Stack

  • HTML5 & CSS3: Responsive UI using semantic HTML and custom CSS grid/flexbox layout.
  • JavaScript: Core application logic with dynamic DOM rendering, event handling, and state management.
  • HLS.js: Embedded for seamless .m3u8 streaming across non-native browsers.
  • EPG Engine: XML parsing, matching, and rendering handled via epg.js.
  • Playlist Engine: Smart parsing and URL validation from local files and remote sources via scripts.js.

Use Cases

Lampone can be used in a wide variety of contexts:

  • Home Media Centers: Serve your own IPTV playlists locally for household viewing.
  • Hospitality: Embed a custom version for hotel or in-room streaming menus.
  • Newsrooms: Create curated channel boards for internal editorial use.
  • Education: Load educational livestreams or remote learning broadcasts.
  • Testing: A perfect sandbox for testing and benchmarking HLS streams.

Browser Compatibility

Lampone relies on the MediaSource Extensions API and HTML5 <video> support for playback. It supports modern browsers that can decode video/mp4 MIME types using HLS.js.

  • ✔ Chrome 39+ (Android & Desktop)
  • ✔ Firefox 41+ (Android), 42+ (Desktop)
  • ✔ Microsoft Edge (Windows 10+)
  • ✔ Safari 9+ (macOS), Safari iPadOS 13+, Safari iOS 17.1+ (via MMS)

Browsers without MediaSource or native HLS support (e.g. old Internet Explorer, some embedded webviews) are not supported.

Tip: Use feature detection to fall back to native playback when possible (e.g. on Safari).

Refer to the MDN MediaSource compatibility matrix.

Audio/Video Codecs

Lampone supports all codecs compatible with HLS.js, including:

  • Video: H.264 (AVC), H.265 (HEVC, in fmp4), VP9
  • Audio: AAC, MP3 (MPEG-1/2 Audio Layer III), AC-3 (Dolby Digital)
  • Containers: MPEG-TS, fMP4, AAC-only, MPEG Audio streams

HLS.js transmuxes these formats into MP4 using Web Workers when available, ensuring compatibility across browsers.

How Lampone Works

Once a user loads a playlist, Lampone parses it line-by-line, extracting metadata such as channel name, logo, group-title, and stream URL. This data is organized into internal structures that feed the sidebar and the player. If EPG is enabled, it attempts to match each stream to its program guide entry and displays Now/Next info under the video element.

Playback relies on HLS.js, which loads the video segments in chunks, assembles them into a continuous buffer, and handles playback logic. Users can toggle stream quality manually if alternate levels are available, or let Lampone manage it based on connection health and device resolution.

Supported M3U Playlist Format

Each playlist must follow the standard #EXTM3U format. Here's a valid example:

#EXTM3U
#EXTINF:-1 tvg-id="Big Buck Bunny" tvg-name="Big Buck Bunny" tvg-logo="" group-title="Sample Videos",Big Buck Bunny
https://test-streams.mux.dev/x36xhzz/x36xhzz.m3u8

The player uses tvg-id or tvg-name to match EPG data, and group-title to organize channels in the sidebar.

Quick Start

git clone https://gitlab.com/lampone-dev/lampone-ui.git
cd lampone-ui
python -m http.server 8000

Or use http-server (Node.js), XAMPP, WAMP, or any HTTP-capable server.
